## 1. 概述

Merkle Patricia Trie (MPT) 是以太坊用于存储世界状态的核心数据结构。它结合了三者的优势：

| 数据结构 | 贡献 |
|---------|------|
| Merkle Tree | 密码学哈希证明，确保数据完整性 |
| Patricia Trie | 压缩公共前缀，节省存储空间 |
| Radix Trie | 16 叉树结构，高效路径查找 |

```
                    ┌─────────────────────────────────────────┐
                    │        Merkle Patricia Trie             │
                    ├─────────────────────────────────────────┤
                    │  Merkle Tree  →  哈希证明、数据完整性    │
                    │  Patricia Trie →  公共前缀压缩           │
                    │  Radix Trie   →  16 叉树、高效查找       │
                    └─────────────────────────────────────────┘
```

### 在以太坊中的应用

```
┌─────────────────────────────────────────────────────────────┐
│                       区块头 (Block Header)                  │
├─────────────────────────────────────────────────────────────┤
│  stateRoot:          Keccak256(世界状态树的根)                │
│  transactionsRoot:   Keccak256(交易树的根)                    │
│  receiptsRoot:       Keccak256(收据树的根)                    │
└─────────────────────────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────────────────────────┐
│                    世界状态树 (State Trie)                    │
├─────────────────────────────────────────────────────────────┤
│  键: Keccak256(地址)                                         │
│  值: RLP 编码的账户信息 {nonce, balance, storageRoot, codeHash}│
└─────────────────────────────────────────────────────────────┘
```

## 2. 数据结构

### 2.1 节点类型

MPT 包含五种节点类型：

```
                    ┌─────────────┐
                    │    Node     │
                    │   (接口)    │
                    └──────┬──────┘
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
        ▼                  ▼                  ▼
┌───────────────┐  ┌───────────────┐  ┌───────────────┐
│   EmptyNode   │  │   LeafNode    │  │ ExtensionNode │
│   (空节点)     │  │   (叶子节点)   │  │   (扩展节点)   │
└───────────────┘  └───────────────┘  └───────────────┘
                                        │
                    ┌───────────────────┼───────────────────┐
                    │                   │                   │
                    ▼                   ▼                   ▼
            ┌───────────────┐  ┌───────────────┐  ┌───────────────┐
            │  BranchNode   │  │   HashNode    │  │   (递归)      │
            │  (分支节点)    │  │  (哈希节点)    │  │               │
            └───────────────┘  └───────────────┘  └───────────────┘
```

#### EmptyNode（空节点）

```
┌─────────────────┐
│    EmptyNode    │
│     无字段      │
└─────────────────┘

哈希值 = Keccak256(RLP("")) = 0x56e81f171bcc55a6...
```

#### LeafNode（叶子节点）

```
┌──────────────────────────────┐
│         LeafNode             │
├──────────────────────────────┤
│  Path:  []byte  ← 剩余路径    │
│  Value: []byte  ← 存储的值    │
└──────────────────────────────┘

RLP 编码: [HP_Encode(Path), Value]
哈希: Keccak256(RLP 编码) 或 直接内联（< 32 字节）
```

#### ExtensionNode（扩展节点）

```
┌──────────────────────────────┐
│       ExtensionNode          │
├──────────────────────────────┤
│  Path: []byte   ← 共享路径    │
│  Next: Node     ← 下一个节点  │
└──────────────────────────────┘

RLP 编码: [HP_Encode(Path), Hash(Next)]
```

#### BranchNode（分支节点）

```
┌──────────────────────────────────────────────────────────┐
│                     BranchNode                           │
├──────────────────────────────────────────────────────────┤
│  Children[16]: [Node]  ← 16 个子节点 (索引 0-f)          │
│  Value:      []byte    ← 可选值（键在此结束时存储）        │
└──────────────────────────────────────────────────────────┘

RLP 编码: [Hash(C0), Hash(C1), ..., Hash(C15), Value]
```

#### HashNode（哈希节点）

```
┌──────────────────────────────┐
│         HashNode             │
├──────────────────────────────┤
│  HashValue: [32]byte ← 子节点│
│               的 Keccak256   │
│               哈希           │
└──────────────────────────────┘

作用: 当子节点 RLP 编码 >= 32 字节时，
      只存储哈希而不是完整节点，节省空间
```

### 2.2 整体结构图

以下示例展示了插入键值对 `0x1234→Alice`、`0x1235→Bob`、`0x5678→Charlie` 后的 Trie 结构：

```
                           ┌─────────────────────────────────────┐
                           │           BranchNode (根)            │
                           │  Children[16] + Value               │
                           └──────────────┬──────────────────────┘
                                          │
                        ┌─────────────────┼─────────────────┐
                        │                 │                 │
                    索引 [1]          索引 [5]         其他索引 (空)
                        │                 │
                        ▼                 ▼
              ┌──────────────────┐  ┌──────────────────┐
              │  ExtensionNode   │  │    LeafNode      │
              │  Path: [0,2,0,3] │  │  Path: [0,6,0,7, │
              │                  │  │                 0,8] │
              └────────┬─────────┘  │  Value: Charlie  │
                       │           └──────────────────┘
                       ▼
              ┌─────────────────────────────────────┐
              │           BranchNode                │
              │  Children[16] + Value               │
              └──────────────┬──────────────────────┘
                             │
              ┌──────────────┼──────────────┐
              │              │              │
          索引 [4]       索引 [5]      其他索引 (空)
              │              │
              ▼              ▼
     ┌──────────────┐ ┌──────────────┐
     │  LeafNode    │ │  LeafNode    │
     │  Path: []    │ │  Path: []    │
     │  Value:Alice │ │  Value: Bob  │
     └──────────────┘ └──────────────┘
```

**路径解释**：
- 键 `0x1234` → nibbles `[1,2,3,4]` → 根分支索引 [1] → 扩展路径 [2,3] → 分支索引 [4] → 叶子 Alice
- 键 `0x1235` → nibbles `[1,2,3,5]` → 根分支索引 [1] → 扩展路径 [2,3] → 分支索引 [5] → 叶子 Bob
- 键 `0x5678` → nibbles `[5,6,7,8]` → 根分支索引 [5] → 叶子 Charlie（剩余路径 [6,7,8]）

## 3. 路径编码

### 3.1 Nibble 编码

将每个字节拆分为两个 4-bit 值（nibble），这是 MPT 所有操作的基础。

```
字节:     0x12        0x34
        ┌──────┐    ┌──────┐
        │00010010│    │00110100│
        └──┬──┬─┘    └──┬──┬─┘
           │  │          │  │
           ▼  ▼          ▼  ▼
nibble:   1  2          3  4

结果: [1, 2, 3, 4]
```

```go
// bytesToNibbles 将字节切片转换为 nibble 切片
func bytesToNibbles(data []byte) []byte {
    nibbles := make([]byte, len(data)*2)
    for i, b := range data {
        nibbles[i*2]   = b >> 4      // 高 4 位
        nibbles[i*2+1] = b & 0x0f    // 低 4 位
    }
    return nibbles
}
```

### 3.2 HP 编码（Hex-Prefix Encoding）

HP 编码用于将 nibble 路径转换为字节，同时标识节点类型（叶子/扩展）和路径长度奇偶性。

#### 编码规则

```
flag = 2 * isLeaf + (len(path) % 2)

isLeaf = true  → flag += 2
isLeaf = false → flag += 0

len(path) 偶数 → flag += 0
len(path) 奇数 → flag += 1
```

#### 四种情况

```
┌──────────────────────────────────────────────────────────────────────┐
│                        HP 编码规则                                    │
├────────────┬────────┬────────┬───────────────────────────────────────┤
│  节点类型   │ 长度   │ Flag   │ 编码格式                              │
├────────────┼────────┼────────┼───────────────────────────────────────┤
│ 扩展节点    │ 偶数   │ 0x00   │ [0x00, 0x00, p0, p1, p2, ...]        │
│ 扩展节点    │ 奇数   │ 0x10   │ [0x10|p0, p1, p2, ...]               │
│ 叶子节点    │ 偶数   │ 0x20   │ [0x20, 0x00, p0, p1, p2, ...]        │
│ 叶子节点    │ 奇数   │ 0x30   │ [0x30|p0, p1, p2, ...]               │
└────────────┴────────┴────────┴───────────────────────────────────────┘
```

#### 编码示例

```
示例 1: 扩展节点，路径 [2,3,4,5]（偶数长度）
  flag = 2*0 + 0 = 0
  编码: [0x00, 0x23, 0x45]

示例 2: 扩展节点，路径 [1,2,3]（奇数长度）
  flag = 2*0 + 1 = 1
  编码: [0x11, 0x23]

示例 3: 叶子节点，路径 [0,6,0,7,0,8]（偶数长度）
  flag = 2*1 + 0 = 2
  编码: [0x20, 0x06, 0x07, 0x08]

示例 4: 叶子节点，路径 [6,0,7,0,8]（奇数长度）
  flag = 2*1 + 1 = 3
  编码: [0x36, 0x07, 0x08]
```

```go
// hexPrefixEncode 实现 HP 编码
func hexPrefixEncode(path []byte, isLeaf bool) []byte {
    flag := 0
    if isLeaf { flag += 2 }
    if len(path)%2 == 1 { flag += 1 }

    result := make([]byte, 0, (len(path)+2)/2+1)

    if flag >= 2 {
        // 奇数长度：第一个字节包含 flag 和第一个 nibble
        if len(path) > 0 {
            result = append(result, byte(flag<<4)|path[0])
            path = path[1:]
        } else {
            result = append(result, byte(flag<<4))
        }
    } else {
        // 偶数长度：第一个字节是 flag
        result = append(result, byte(flag<<4))
    }

    // 将剩余 nibbles 两两组合
    for i := 0; i < len(path); i += 2 {
        if i+1 < len(path) {
            result = append(result, path[i]<<4|path[i+1])
        } else {
            result = append(result, path[i]<<4)
        }
    }

    return result
}
```

## 4. 哈希机制

### 4.1 Keccak256

以太坊使用 **Keccak256** 哈希算法（注意：不是标准的 SHA3-256，而是其前身）。

```
                    ┌──────────────┐
                    │   Keccak256  │
                    └──────┬───────┘
                           │
          ┌────────────────┼────────────────┐
          │                │                │
          ▼                ▼                ▼
    ┌──────────┐    ┌──────────┐    ┌──────────┐
    │ RLP 编码  │    │ 节点数据  │    │  32 字节  │
    │ 的节点    │ ──▶│ 哈希计算  │ ──▶│  哈希值   │
    └──────────┘    └──────────┘    └──────────┘
```

```go
// sha3Keccak256 计算 Keccak256 哈希
func sha3Keccak256(data []byte) []byte {
    h := sha3.NewLegacyKeccak256()
    h.Write(data)
    return h.Sum(nil)
}
```

### 4.2 内联节点规则

```
┌─────────────────────────────────────────────────────────────┐
│                     节点存储规则                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  RLP 编码长度 < 32 字节                                      │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  直接内联存储                                         │   │
│  │  父节点直接包含子节点的完整 RLP 编码                     │   │
│  │  优点: 减少数据库查询，提高读取速度                      │   │
│  └──────────────────────────────────────────────────────┘   │
│                                                             │
│  RLP 编码长度 >= 32 字节                                     │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  存储 Keccak256 哈希                                  │   │
│  │  父节点只存储子节点的 32 字节哈希                        │   │
│  │  优点: 节省存储空间，支持增量同步                        │   │
│  └──────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

```go
// hashNode 计算节点哈希
func hashNode(encoded []byte) []byte {
    if len(encoded) < 32 {
        // 短节点直接内联
        return encoded
    }
    // 长节点计算 Keccak256 哈希
    return sha3Keccak256(encoded)
}
```

## 5. 核心算法

### 5.1 插入算法

插入算法是 MPT 最复杂的操作，需要处理四种节点类型的不同情况。

#### 算法流程图

```
                    ┌─────────────┐
                    │  Insert(k,v)│
                    └──────┬──────┘
                           ▼
                    ┌─────────────┐
                    │ 当前节点类型 │
                    └──────┬──────┘
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
        ▼                  ▼                  ▼
   ┌──────────┐      ┌──────────┐      ┌──────────┐
   │ EmptyNode│      │ LeafNode │      │ BranchNode│
   └────┬─────┘      └────┬─────┘      └────┬─────┘
        │                 │                 │
        ▼                 ▼                 ▼
   ┌──────────┐    ┌──────────────┐   ┌──────────────┐
   │创建新叶子 │    │计算公共前缀   │   │根据第一个    │
   │节点      │    │根据匹配情况   │   │nibble 选择   │
   │          │    │创建分支或    │   │子节点递归    │
   │          │    │更新值        │   │插入          │
   └──────────┘    └──────────────┘   └──────────────┘
```

#### 详细插入逻辑

```
情况 A: 空节点
  ┌─────────────────────────────────────────┐
  │  直接创建 LeafNode{Path: key, Value: v}  │
  └─────────────────────────────────────────┘

情况 B: 叶子节点
  计算公共前缀长度 commonLen

  ┌─────────────────────────────────────────────────────────────┐
  │                                                             │
  │  B1: key == leaf.Path (完全匹配)                            │
  │      → 更新值: LeafNode{Path: leaf.Path, Value: v}           │
  │                                                             │
  │  B2: 部分匹配 (commonLen > 0)                               │
  │      → 创建 ExtensionNode                                    │
  │        Path: leaf.Path[:commonLen]                          │
  │        Next: BranchNode                                      │
  │          Children[leafRemaining[0]] = Leaf{剩余路径, 原值}    │
  │          Children[keyRemaining[0]] = Leaf{剩余路径, 新值}     │
  │                                                             │
  │  B3: 完全不匹配 (commonLen == 0)                             │
  │      → 创建 BranchNode                                      │
  │        Children[leaf.Path[0]] = Leaf{剩余路径, 原值}         │
  │        Children[key[0]] = Leaf{剩余路径, 新值}               │
  │                                                             │
  └─────────────────────────────────────────────────────────────┘

情况 C: 扩展节点
  计算公共前缀长度 commonLen

  ┌─────────────────────────────────────────────────────────────┐
  │                                                             │
  │  C1: key 包含整个扩展路径 (commonLen == len(ext.Path))       │
  │      → 递归插入到 Next 节点                                  │
  │        ExtensionNode{Path: ext.Path, Next: Insert(Next,...)} │
  │                                                             │
  │  C2: key 只匹配部分扩展路径                                  │
  │      → 分裂扩展节点，创建 BranchNode                         │
  │        处理原扩展路径剩余部分                                 │
  │        处理新键剩余部分                                       │
  │        如果有公共前缀，用 ExtensionNode 包装                  │
  │                                                             │
  └─────────────────────────────────────────────────────────────┘

情况 D: 分支节点
  ┌─────────────────────────────────────────────────────────────┐
  │                                                             │
  │  D1: key 已用完 (len(key) == 0)                              │
  │      → 更新分支节点的值                                      │
  │        BranchNode{Value: v, Children: ...}                   │
  │                                                             │
  │  D2: key 未用完                                             │
  │      → 根据第一个 nibble 选择子节点                          │
  │        idx = key[0]                                         │
  │        Children[idx] = Insert(Children[idx], key[1:], v)     │
  │                                                             │
  └─────────────────────────────────────────────────────────────┘
```

```go
// insert 主函数
func (t *Trie) insert(node Node, key, value []byte) Node {
    if node == nil {
        return &LeafNode{Path: key, Value: value}
    }
    if _, ok := node.(*EmptyNode); ok {
        return &LeafNode{Path: key, Value: value}
    }

    switch n := node.(type) {
    case *LeafNode:
        return t.insertIntoLeaf(n, key, value)
    case *ExtensionNode:
        return t.insertIntoExtension(n, key, value)
    case *BranchNode:
        return t.insertIntoBranch(n, key, value)
    case *HashNode:
        return &LeafNode{Path: key, Value: value}
    }
    return node
}
```

#### 插入示例

插入 `0x1234 → Alice`、`0x1235 → Bob`、`0x5678 → Charlie` 的过程：

```
步骤 1: 插入 0x1234 → Alice

  空 Trie
       │
       ▼
  创建 LeafNode{Path: [1,2,3,4], Value: "Alice"}


步骤 2: 插入 0x1235 → Bob

  当前根: LeafNode{[1,2,3,4], "Alice"}
  新键:   [1,2,3,5]

  公共前缀: [1,2,3] (长度 3)

  分裂为:
  ExtensionNode{
    Path: [1,2,3]
    Next: BranchNode{
      Children[4]: LeafNode{[], "Alice"}
      Children[5]: LeafNode{[], "Bob"}
    }
  }


步骤 3: 插入 0x5678 → Charlie

  当前根: ExtensionNode{...}
  新键:   [5,6,7,8]

  公共前缀: [] (长度 0)

  需要创建新的根分支节点:
  BranchNode{
    Children[1]: ExtensionNode{Path: [2,3], Next: Branch{...}}
    Children[5]: LeafNode{[6,7,8], "Charlie"}
  }
```

### 5.2 查询算法

查询算法从根节点开始，根据键的 nibbles 逐层向下查找。

```
                    ┌─────────────┐
                    │  Get(key)   │
                    └──────┬──────┘
                           ▼
                    ┌─────────────┐
                    │ 当前节点类型 │
                    └──────┬──────┘
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
        ▼                  ▼                  ▼
   ┌──────────┐      ┌──────────┐      ┌──────────┐
   │ EmptyNode│      │ LeafNode │      │ BranchNode│
   └────┬─────┘      └────┬─────┘      └────┬─────┘
        │                 │                 │
        ▼                 ▼                 ▼
   ┌──────────┐    ┌──────────────┐   ┌──────────────┐
   │ 返回未   │    │ 比较 Path     │   │ 取 key[0]    │
   │ 找到     │    │ 完全匹配?     │   │ 作为索引     │
   │          │    │ 是→返回值     │   │ 递归查询     │
   │          │    │ 否→未找到     │   │ 剩余 key[1:] │
   └──────────┘    └──────────────┘   └──────────────┘
```

```go
// get 递归查询
func (t *Trie) get(node Node, key []byte) ([]byte, bool) {
    switch n := node.(type) {
    case *EmptyNode:
        return nil, false

    case *LeafNode:
        if bytes.Equal(n.Path, key) {
            return n.Value, true
        }
        return nil, false

    case *ExtensionNode:
        prefixLen := commonPrefixLength(n.Path, key)
        if prefixLen < len(n.Path) {
            return nil, false  // 前缀不匹配
        }
        return t.get(n.Next, key[len(n.Path):])

    case *BranchNode:
        if len(key) == 0 {
            if n.Value != nil {
                return n.Value, true
            }
            return nil, false
        }
        idx := key[0]
        child := n.Children[idx]
        if child == nil {
            return nil, false
        }
        return t.get(child, key[1:])
    }
    return nil, false
}
```

### 5.3 更新算法

更新算法与插入算法相同：

```
┌─────────────────────────────────────────────────────────────┐
│                     更新逻辑                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Insert(key, newValue)                                      │
│       │                                                     │
│       ▼                                                     │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  1. 找到匹配的键 → 更新值                              │   │
│  │  2. 未找到 → 插入新键值对                              │   │
│  └──────────────────────────────────────────────────────┘   │
│                                                             │
│  注意: MPT 是不可变的，更新会创建新节点                        │
│        旧节点保持不变                                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.4 删除算法

删除操作需要处理节点合并以保持 MPT 的压缩特性。

```
                    ┌─────────────┐
                    │  Delete(k)  │
                    └──────┬──────┘
                           ▼
                    ┌─────────────┐
                    │ 找到目标节点 │
                    └──────┬──────┘
                           ▼
                    ┌─────────────┐
                    │ 删除节点     │
                    └──────┬──────┘
                           ▼
                    ┌─────────────┐
                    │ 检查是否需要 │
                    │ 合并节点     │
                    └──────┬──────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
              ▼            ▼            ▼
         ┌────────┐  ┌──────────┐ ┌──────────┐
         │分支剩 1 │  │扩展下一个│ │其他情况  │
         │个子节点│  │为空     │ │保持不变  │
         └───┬────┘  └────┬─────┘ └──────────┘
             │            │
             ▼            ▼
         ┌────────┐  ┌──────────┐
         │合并为  │  │删除扩展  │
         │扩展或  │  │节点     │
         │叶子    │  │         │
         └────────┘  └──────────┘
```

```go
// delete 递归删除
func (t *Trie) delete(node Node, key []byte) (Node, bool) {
    switch n := node.(type) {
    case *LeafNode:
        if bytes.Equal(n.Path, key) {
            return &EmptyNode{}, true  // 找到并删除
        }
        return node, false

    case *ExtensionNode:
        prefixLen := commonPrefixLength(n.Path, key)
        if prefixLen < len(n.Path) {
            return node, false
        }
        newNext, deleted := t.delete(n.Next, key[len(n.Path):])
        if !deleted {
            return node, false
        }
        return t.mergeExtension(n.Path, newNext), true

    case *BranchNode:
        // ... 递归删除子节点 ...
        return t.mergeBranch(newBranch), true
    }
    return node, false
}
```

### 5.5 节点合并算法

#### mergeBranch：分支节点合并

```
┌─────────────────────────────────────────────────────────────┐
│                  mergeBranch 规则                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  条件: 分支节点只剩 1 个子节点 且 没有值                      │
│                                                             │
│  操作:                                                       │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  子节点是 LeafNode:                                   │   │
│  │    → 合并为新 LeafNode                                │   │
│  │      Path: [lastIdx] + child.Path                    │   │
│  │      Value: child.Value                              │   │
│  │                                                      │   │
│  │  子节点是 ExtensionNode:                              │   │
│  │    → 合并为新 ExtensionNode                           │   │
│  │      Path: [lastIdx] + child.Path                    │   │
│  │      Next: child.Next                                │   │
│  │                                                      │   │
│  │  其他:                                                │   │
│  │    → 创建 ExtensionNode{Path: [lastIdx], Next: child} │   │
│  └──────────────────────────────────────────────────────┘   │
│                                                             │
│  否则: 保持分支节点不变                                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### mergeExtension：扩展节点合并

```
┌─────────────────────────────────────────────────────────────┐
│                  mergeExtension 规则                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  条件: 删除后扩展节点的 Next 指向空节点                        │
│                                                             │
│  操作:                                                       │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  Next 是 EmptyNode:                                   │   │
│  │    → 返回 EmptyNode                                   │   │
│  │                                                      │   │
│  │  Next 也是 ExtensionNode:                             │   │
│  │    → 合并路径                                         │   │
│  │      Path: path + next.Path                          │   │
│  │      Next: next.Next                                 │   │
│  │                                                      │   │
│  │  Next 是 LeafNode:                                    │   │
│  │    → 合并为 LeafNode                                  │   │
│  │      Path: path + leaf.Path                          │   │
│  │      Value: leaf.Value                               │   │
│  └──────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 6. Merkle Proof

### 6.1 证明生成

Merkle Proof 允许在不下载完整 Trie 的情况下验证某个键值对的存在性。

```
┌─────────────────────────────────────────────────────────────┐
│                    证明生成过程                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  GenerateProof(key):                                        │
│       │                                                     │
│       ▼                                                     │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  1. 从根节点开始                                      │   │
│  │  2. 逐层向下遍历，将路径上的每个节点添加到证明列表      │   │
│  │  3. 到达叶子节点后，检查路径是否匹配                   │   │
│  │  4. 如果匹配，记录值                                  │   │
│  └──────────────────────────────────────────────────────┘   │
│                                                             │
│  证明内容:                                                   │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  Proof {                                             │   │
│  │    Key:   要证明的键                                  │   │
│  │    Nodes: 从根到叶子的节点路径（RLP 编码）             │   │
│  │    Value: 键对应的值（如果存在）                       │   │
│  │  }                                                    │   │
│  └──────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

```
证明路径示意图:

         Root
          │
          ▼
    ┌──────────┐
    │ Node 0   │  ← 根节点（添加到证明）
    │ (Branch) │
    └────┬─────┘
         │
         ▼
    ┌──────────┐
    │ Node 1   │  ← 扩展节点（添加到证明）
    │(Extension)│
    └────┬─────┘
         │
         ▼
    ┌──────────┐
    │ Node 2   │  ← 叶子节点（添加到证明）
    │  (Leaf)  │
    └──────────┘

Proof = {Key, [Node0_RLP, Node1_RLP, Node2_RLP], Value}
```

### 6.2 证明验证

```
┌─────────────────────────────────────────────────────────────┐
│                    证明验证过程                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  VerifyProof(rootHash, proof):                              │
│       │                                                     │
│       ▼                                                     │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  1. 计算第一个节点（根节点）的哈希                     │   │
│  │  2. 与已知的 rootHash 比较                           │   │
│  │  3. 逐层验证：检查每个子节点哈希是否被父节点引用        │   │
│  │  4. 如果所有验证通过，返回值                          │   │
│  └──────────────────────────────────────────────────────┘   │
│                                                             │
│  验证成功意味着:                                             │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  ✓ 数据存在于具有该 rootHash 的 Trie 中                │   │
│  │  ✓ 数据未被篡改                                       │   │
│  │  ✓ 不需要下载完整 Trie                                │   │
│  └──────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

```go
// VerifyProof 验证 Merkle Proof
func VerifyProof(rootHash []byte, proof *Proof) (bool, []byte) {
    if len(proof.Nodes) == 0 {
        return false, nil
    }

    // 验证根节点哈希
    firstNode := proof.Nodes[0]
    firstHash := hashNode(firstNode)
    if !bytes.Equal(firstHash, rootHash) {
        if len(firstNode) >= 32 {
            return false, nil
        }
    }

    // 逐层验证节点哈希关系
    for i := 1; i < len(proof.Nodes); i++ {
        node := proof.Nodes[i]
        parentNode := proof.Nodes[i-1]
        nodeHash := hashNode(node)

        // 检查子节点哈希是否被父节点引用
        if !bytes.Contains(parentNode, nodeHash) {
            if len(node) >= 32 {
                return false, nil
            }
        }
    }

    return true, proof.Value
}
```

## 7. 复杂度分析

| 操作 | 时间复杂度 | 空间复杂度 | 说明 |
|-----|-----------|-----------|------|
| 插入 | O(log n) | O(log n) | n 为键值对数量，每次插入创建新路径 |
| 查询 | O(log n) | O(1) | 只需遍历从根到叶子的路径 |
| 更新 | O(log n) | O(log n) | 与插入相同，创建新节点 |
| 删除 | O(log n) | O(log n) | 可能需要合并节点 |
| 生成证明 | O(log n) | O(log n) | 收集路径上的节点 |
| 验证证明 | O(log n) | O(1) | 逐层验证哈希 |

### 证明大小

| Trie 深度 | 节点数 | 证明大小（估算） |
|----------|--------|-----------------|
| 3        | 3      | ~450 B          |
| 5        | 5      | ~750 B          |
| 7        | 7      | ~1.1 KB         |
| 10       | 10     | ~1.5 KB         |

典型以太坊状态树深度为 3-7，证明大小通常 < 3KB。
