# 比特币工作量证明（Proof of Work, PoW）

## 概述

本项目是比特币工作量证明（PoW）机制的学习实现，使用 Go 语言编写，包含详细的注释和交互式演示。

## 核心概念

### 什么是工作量证明？

工作量证明（Proof of Work）是比特币网络中用于达成共识的算法。矿工通过消耗算力来解决一个数学难题，找到满足特定条件的区块哈希值。

```
挖矿 = 寻找一个 Nonce 值，使得 SHA256(SHA256(区块头)) < 目标值
```

### 比特币区块头结构（80字节）

```
┌──────────────┬─────────────────────┬─────────────────────┐
│  4字节       │     32字节          │     32字节          │
│  版本号       │  前一区块哈希        │  Merkle树根哈希      │
├──────────────┼─────────────────────┼─────────────────────┤
│  4字节       │     4字节           │     4字节           │
│  时间戳       │  难度目标(Bits)      │  Nonce             │
└──────────────┴─────────────────────┴─────────────────────┘
```

### 难度机制

难度通过要求哈希值的**前导零数量**来控制：

```
难度 = 1  →  哈希以 "0"    开头（期望尝试 16 次）
难度 = 2  →  哈希以 "00"   开头（期望尝试 256 次）
难度 = 3  →  哈希以 "000"  开头（期望尝试 4,096 次）
难度 = 4  →  哈希以 "0000" 开头（期望尝试 65,536 次）
...
每增加 1 个难度，工作量约增加 16 倍
```

### 为什么需要双重 SHA256？

比特币使用 `SHA256(SHA256(data))` 而非单次 SHA256：

- **防止长度扩展攻击**：某些哈希函数（如 SHA256）存在长度扩展漏洞
- **增加安全性**：即使 SHA256 被发现弱点，双重哈希提供额外保护

## 挖矿 vs 验证

这是 PoW 的核心特性：

```
挖矿（困难）:                    验证（简单）:
┌─────────────────┐             ┌─────────────────┐
│ 尝试 Nonce=0    │             │ 计算一次哈希      │
│ 计算哈希        │             │ 检查前导零数量    │
│ 不满足? → 继续  │   ═══►      │ 满足? → 有效      │
│ 尝试 Nonce=1    │             │ 不满足? → 无效    │
│ 计算哈希        │             └─────────────────┘
│ ...             │                  ~1次哈希计算
│ 尝试 Nonce=N    │
│ 找到了!         │
│                 │
│ 约10万次哈希计算 │
└─────────────────┘
```

## 区块链链接原理

```
创世区块                          第二个区块
┌─────────────────┐              ┌─────────────────┐
│ PrevHash: 000... │              │ PrevHash: 00ab  │ ◄── 创世区块的哈希
│ Hash:    00ab... │ ─── 链接 ──► │ Hash:    00ef... │
│ Nonce:   418     │              │ Nonce:   114     │
└─────────────────┘              └─────────────────┘
```

每个区块的 `PrevHash` 指向前一个区块的哈希值，形成不可篡改的链。如果修改了创世区块的数据，其哈希值会改变，导致第二个区块的 `PrevHash` 不再匹配，需要重新计算第二个区块的 PoW。

## 比特币难度对比

| 时期 | 难度 | 前导零 | 期望尝试次数 |
|------|------|--------|-------------|
| 创世区块 (2009) | 1 | ~1个 | 16 |
| 本项目演示 | 2-4 | 2-4个 | 256-65,536 |
| 比特币 2024 | ~83万亿 | ~18个 | 8.3×10^13 |

比特币网络的难度会每 2016 个区块（约 2 周）自动调整一次，目标是保持平均 10 分钟出一个区块。
